# Cloudflare Pages + D1 기반 모바일 SMS 본인인증 투표 시스템
## SaaS 비즈니스 모델 및 Niche 시장 침투 로드맵

> **현황 요약**: 현재는 20~500명 규모의 회원 안건 투표를 Cloudflare Pages + KV로 처리하는 경량 솔루션입니다. 본 문서는 이를 D1 기반 멀티테넌트 SaaS로 전환하고, Niche 시장을 중심으로 유료화·수익 극대화를 달성하는 구체적인 전략입니다.

---

## 1. Executive Summary

| 항목 | 내용 |
|------|------|
| **제품 본질** | "모바일 1:1 링크 + 비밀번호(SMS/카톡) 기반 익명 안건 투표"를 5분 만에 생성·배포·집계하는 경량 SaaS |
| **핵심 가치** | 법적·행정적 효력이 필요 없지만 **공정성·익명성·편의성**이 필요한 조직의 의사결정 속도 향상 |
| **수익 모델** | **1회 투표 건당 과금** (최소인원 기본료 + 초과 인원 종량제) + 부가 기능(브랜딩·감사로그·API·SMS 발송) |
| **목표 Niche** | 공동주택 입주자대표회의/아파트 관리위원회 → 주민자치회 → 동창회/동호회 → 중소기업 임직원 |
| **1년 목표** | 100개 유료 테넌트 확보, 누적 매출 ₩50M 달성 |
| **3년 목표** | 공동주택·주민자치 시장 20% 점유, 연 매출 ₩500M + 부가서비스 30% 비중 |

## 2. 제품 분석 및 포지셔닝

### 2.1 현재 강점
- **배포가 쉬움**: Cloudflare Pages + KV/D1으로 거의 묣지 않은 운영 비용
- **보안 설계 우수**: 익명 투표용지 분리, POST 기반 비밀번호 전송, 상수 시간 비교
- **모바일 최적화**: 인앱브라우저 우회, Pretendard 폰트, 터치 친화 UI
- **간편한 관리**: 관리자 페이지에서 안건/회원/디자인/결과 실시간 관리

### 2.2 현재 한계 (SaaS 전환 시 해결 과제)
- **단일 테넌트**: 프로젝트 당 하나의 투표만 운영 가능
- **수동 회원 등록**: CSV 업로드·연동 기능 부재
- **SMS 미지원**: 카카오톡 메시지 수동 복사에 의존
- **감사 추적 부재**: 누가 언제 관리했는지 기록 없음
- **결제·청구 시스템 없음**

### 2.3 포지셔닝
> **"모바일로 5분 만에 여는 익명 전자투표함"**
> 
> 정관·회칙상 "서면 투표"를 대체할 수는 없지만, **사전 경선·임시 의결·동의·설문·경선** 등 비공식적이면서도 공정성이 필요한 모든 의사결정에 사용된다.

## 3. 비즈니스 모델 (Business Model)

### 3.1 가치 사슬

```
[관리자] 안건 입력 + 회원 등록 ──→ [SaaS] 링크/비밀번호 생성 ──→ [회원] SMS/카톡 수신
                                                          ↓
[관리자] 실시간 집계 확인 ←────────── [SaaS] 익명 투표 수집 ←─── [회원] 모바일 투표
```

### 3.2 핵심 지표 (North Star Metrics)

| 지표 | 정의 | 목표 |
|------|------|------|
| **TTFV (Time to First Vote)** | 가입 후 첫 투표 개시까지 시간 | 10분 이내 |
| **투표 완료율** | 발송 대비 제출 완료 비율 | 75% 이상 |
| **재사용율** | 이전 투표 이후 다음 투표까지 재구매율 | 50% 이상 |
| **NPS** | 추천 지수 | 50점 이상 |
| **CAC 대비 LTV** | 고객 획득 비용 대비 생애 가치 | 3:1 이상 |

## 4. SaaS 유료화 전략

### 4.1 요금제 구조: 1회 투표 건당 과금 (Pay Per Vote)

투표는 **매달 사용하는 서비스가 아니라 1년에 몇 번만 사용하는 이벤트성 의사결정**입니다. 따라서 월 구독 모델 대신 **"1회 투표 건당 결제"** 모델을 채택합니다.

기본 구조:
- **최소 인원(50명)까지는 정액 기본료**를 받습니다.
- **최소 인원 초과분은 1인당 종량제(₩150~₩200)**로 과금합니다.
- 추가 부가 기능은 별도 옵션으로 선택합니다.

| 플랜 | 기본료(1회) | 포함 인원 | 초과 인원 단가 | 주요 포함 기능 | 타겟 |
|------|------------|-----------|----------------|----------------|------|
| **Free** | 묣음 | 최대 30명 | - | 1회 투표, 기본 디자인, 7일 데이터 보관, 결과 공개 | 개인·소모임 체험 |
| **Basic** | **₩49,000** | 최대 50명 | - | 1회 투표, CSV 업로드, 브랜드 컬러, 1년 보관 | 동호회·소규모 모임 |
| **Standard** | **₩49,000** + 초과 인원 과금 | 50명 | **₩180/인** | Basic 기능 + SMS 발송(50건), 감사로그, PDF 보고서 | 동창회·주민자치회·중소 단체 |
| **Pro** | **₩99,000** + 초과 인원 과금 | 50명 | **₩150/인** | Standard 기능 + 무제한 SMS(실제 발송 수 과금), API, 커스텀 도메인, 우선 지원 | 공동주택·대형 단체 |
| **Enterprise** | 맞춤 견적 | 50명+ | 협의 | Pro 기능 + SSO, 전용 인프라, 커스텀 계약, 전담 CSM | 대형 아파트·협회·정부기관 |

### 4.2 구체적 시뮬레이션 예시

#### 예시 1: 대형 아파트 단지 (500세대)
- **상황**: 관리규약 개정 사전 동의, 입주자대표회 안건 투표

## 5. 수익 극대화 전략

### 5.1 멀티테넌트 아키텍처 (D1 기반)

```sql
-- tenants 테이블
CREATE TABLE tenants (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  plan TEXT NOT NULL,
  base_fee INTEGER,
  base_members INTEGER,
  per_member_rate INTEGER,
  custom_domain TEXT,
  branding JSON,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- votes, members, ballots 테이블 (tenant_id로 분리)
CREATE TABLE votes (... tenant_id TEXT, ...);
CREATE TABLE members (... tenant_id TEXT, ...);
CREATE TABLE ballots (... vote_id TEXT, ...);
CREATE TABLE audit_logs (... tenant_id TEXT, action TEXT, actor TEXT, ...);
```

- **하나의 D1 데이터베이스**로 모든 테넌트를 분리 관리
- Cloudflare Workers를 통해 인증·인가·과금 집계 처리
- R2에 감사로그·백업 아카이브

### 5.2 핵심 부가가치 기능 (Upsell)

1. **SMS 자동 발송**
   - 회원 등록 시 휴태폰 번호와 함께 자동 SMS 발송
   - 리마인드(미투표자 1회/3일 전/당일) 자동 발송
   - 수익성 높은 과금 아이템

2. **커스텀 브랜딩**
   - 아파트 로고, 관리사무소명, 색상 적용
   - 커스텀 도메인 지원 (vote.아파트명.co.kr)

3. **감사 및 법적 효력 강화**
   - 관리자 행동 로그 (누가 언제 초기화/수정했는지)
   - PDF 결과 보고서 + 전자서명
   - IP/시간 기록 (완전 익명은 아닌 "준익명")

4. **연동 생태계**
   - 카카오톡 채널 연동 (알림톡)
   - 네이버 밴드/카페 공유
   - 아파트 관리 앱/홈페이지 임베드 (iframe/API)

5. **AI 기반 안건 작성 도우미**
   - 안건 문구 다듬기
   - 투표 안남문 자동 생성
   - 추가 과금 가능

### 5.3 잔존율(Churn) 방어

- **온볇ィング 이메일 시퀀스**: 가입 후 7일 동안 5회 발송
- **템플릿 마켓**: "입주자대표회 임시총회", "주민자치회 예산편성", "동창회 회장선거" 등 바로 사용 가능한 템플릿
- **리마인드 자동화**: 투표 마감 3일 전/1일 전/당일 자동 알림
- **성공 사례 공유**: "XX아파트 관리위원회, 투표 완료율 92% 달성"
- **투표 크레딧 유효기간 연장**: 1년 내 미사용 시 6개월 추가 유효기간 부여

## 6. Niche 시장 침투 로드맵

### Phase 1: Beachhead — 공동주택/아파트 (0~6개월)
**이유**
- 의사결정 주체(관리사무소장, 입주자대표회 임원)가 명확
- 투표 빈도가 높음(관리규약 개정, 장기수선계획, 주요 공사, 관리위원회 선출)
- 기존 "서면·방문·현수막" 방식의 불편함이 큼
- 입소문이 빠름(아파트 단지 간 네트워크)

**침투 전략**
1. **파일럿 고객 3~5개 단지 묣음 제공**
   - 조건: 사용 후 인터뷰/후기 영상/성과 데이터 공유 허용
   - 목표: 사례 연구(Case Study) 확보
2. **부동산 관리업체 파트너십**
   - 공동주택 관리업체에 화이트라벨/리셀러 모델 제안
   - 관리사무소장 커뮤니티(카페, 협회) 집중 홍보
3. **지역 커뮤니티 타겟 광고**
   - 네이버 카페 "아파트 관리", "입주자대표회" 등 키워드 광고
   - 지역 박람회(주택관리박람회) 참가

**성과 지표**
- 6개월 내 30개 아파트 단지 도입
- 평균 투표 완료율 70% 이상

---

### Phase 2: 수평 확장 — 주민자치회·마을회·동창회 (6~12개월)
**이유**
- 공동주택 사례를 바탕으로 "지역공동체 의사결정"으로 확장
- 예산 편성, 사업 선정, 대표 선출 등 투표 니즈 명확
- 주민자치회는 정부 예산 지원을 받아 SaaS 비용 부담 적음

**침투 전략**
1. **주민자치회 활동가 양성 교육**
   - "온라인 투표로 주민 참여 2배 높이기" 워크숍
   - 교육 참가자에게 3개월 묣음 이용권 제공
2. **동창회/동호회 템플릿 출시**

## 12. 결론

이 투표 시스템은 기술적으로는 이미 **MVP를 넘어선 상태**입니다. 남은 과제는 **"누가 왜 돈을 내고 쓰는가"**를 명확히 하는 것입니다.

**핵심 전략**은 다음과 같습니다:

1. **Niche 집중**: 공동주택·아파트를 Beachhead로 삼아 신뢰와 사례를 쌓는다.
2. **1회 투표 건당 과금**: 구독형이 아닌 "최소인원 기본료 + 초과 인원 종량제"로 가격 설계한다.
3. **D1 멀티테넌트 전환**: 운영 효율성과 확장성을 확보한다.
4. **SMS + 감사로그 + 브랜딩으로 차별화**: 구글폼·네이버폼과 묣음 경쟁하지 않는다.
5. **성과 기반 업셀**: 투표 완료율, 시간 절감, 관리 편의성, 비용 절감을 수치화해 제시한다.
6. **커뮤니티 + 파트너십**: 직접 영업보다 관리사무소장·주민자치회 활동가를 통해 확산한다.

이 로드맵을 따라 6개월 내 첫 유료 고객 30곳, 1년 내 연 매출 ₩40M은 현실적인 목표입니다.
