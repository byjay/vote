# 회원 안건 투표 사이트 (Cloudflare Pages)

20명 회원에게 개인별 링크 + 비밀번호를 카카오톡으로 보내고, 안건별로 가/부만 투표받는 사이트입니다.
관리자는 제목/문항/디자인(글자크기·색상)을 자유롭게 설정할 수 있고, 회원 명단을 넣으면 랜덤 비밀번호와 개인별 링크가 자동 생성됩니다. 투표는 익명 집계로 처리되어(누가 무엇을 찍었는지는 저장하지 않음) 참여 여부만 관리자가 확인할 수 있습니다.

## 1. 준비물
- Cloudflare 계정 (무료)
- Node.js 설치되어 있어야 함 (wrangler CLI 사용)

## 2. 배포 방법

### 2-1. Wrangler 설치 및 로그인
```
npm install -g wrangler
wrangler login
```

### 2-2. KV 네임스페이스 생성 (회원/설정/결과 데이터 저장용)
```
wrangler kv namespace create VOTE_KV
```
실행하면 출력되는 `id` 값을 복사해서 `wrangler.toml` 파일의
```
id = "REPLACE_WITH_YOUR_KV_ID"
```
부분에 붙여넣으세요.

### 2-3. 관리자 비밀번호 설정
이 값이 관리자 페이지(admin.html) 로그인 비밀번호입니다. 원하는 비밀번호로 바꿔서 실행하세요.
```
wrangler pages project create member-vote
wrangler pages secret put ADMIN_PASSWORD --project-name=member-vote
```
(명령 실행 후 프롬프트에 원하는 관리자 비밀번호를 입력)

### 2-4. 배포
프로젝트 폴더(vote-app) 최상단에서 실행:
```
wrangler pages deploy public --project-name=member-vote
```
배포가 끝나면 `https://member-vote.pages.dev` 같은 주소가 발급됩니다.

> 이후 설정이나 문항을 바꿀 때는 다시 배포할 필요 없이, 배포된 admin.html에서 바로 저장하면 됩니다. 코드 자체를 수정했을 때만 `wrangler pages deploy public --project-name=member-vote`를 다시 실행하세요.

## 3. 사용 순서

1. `https://your-site.pages.dev/admin.html` 접속 → 관리자 비밀번호로 로그인
2. **설정 탭**: 제목, 안내문, 안건(가/부 문항) 입력, 글자크기·강조색상·배경색 설정 후 "설정 저장"
3. **회원 관리 탭**: 회원 이름 20명을 한 줄에 한 명씩 입력 → "추가 생성" 클릭
   → 표에 이름별 비밀번호가 생성되고, 아래 "카카오톡으로 보낼 개인별 메시지" 박스에 각자에게 보낼 메시지(링크+비밀번호)가 자동으로 만들어집니다. "전체 복사" 버튼으로 복사해서 각 회원에게 1:1로 카톡 발송하면 됩니다.
   - 링크 형태: `https://your-site.pages.dev/?id=회원고유ID`
   - 이 ID만으로는 투표할 수 없고, 반드시 본인 비밀번호를 알아야 투표 가능합니다.
4. 회원이 링크 접속 → 비밀번호 입력 → 안건별 가/부 선택 → 제출 (1인 1회만 가능, 재투표 불가)
5. **결과 탭**: 문항별 가/부 집계와 전체 참여 인원(00/20명)을 실시간으로 확인
   - **집계 초기화** 버튼: 회원 명단은 유지한 채 집계와 참여 기록만 초기화 (재투표 진행 시 사용)

## 4. 보안/안정성 설계

- **비밀번호 URL 노출 없음**: 비밀번호는 GET 쿼리스트링이 아닌 POST 본문으로만 전송되어 서버 로그·브라우저 기록에 남지 않습니다.
- **비밀 투표**: 개별 회원이 무엇을 선택했는지는 회원과 연결되지 않는 익명 투표용지(`ballot:*` 키)로만 저장됩니다. 관리자는 "누가 투표를 완료했는지"만 확인 가능합니다.
- **표 유실 방지**: 투표용지를 개별 KV 키에 저장하므로 여러 명이 동시에 제출해도 집계가 유실되지 않습니다(기존 단일 키 누적 방식의 race condition 제거).
- **XSS 방지**: 모든 화면 출력은 `textContent` 기반으로 렌더링되어 문항/이름에 특수문자를 넣어도 안전합니다.
- **입력 검증**: 문항 수(최대 50), 회원 수(최대 500), 텍스트 길이, 색상 형식(hex) 등을 서버에서 검증합니다.
- **타이밍 공격 방어**: 비밀번호 비교는 상수 시간 비교 함수를 사용합니다.
- **비밀번호는 평문으로 저장**됩니다(20인 내부 투표 수준의 가벼운 보안 목적). 회사 인증서 로그인처럼 강한 보안이 필요하면 별도 강화가 필요합니다.
- 무료 Cloudflare Pages + KV 사용량으로 20명 규모는 충분히 처리됩니다.

### KV 데이터 구조
| 키 | 내용 |
|---|---|
| `config` | 제목/문항/디자인 설정 |
| `members` | 회원 목록 (id, 이름, 비밀번호) |
| `voted:{회원id}` | 해당 회원의 투표 완료 여부 (내용은 저장 안 함) |
| `ballot:{랜덤}` | 익명 투표용지 (회원과 연결 불가) |

## 5. 파일 구조
```
vote-app/
├── wrangler.toml
├── public/
│   ├── index.html      # 회원 투표 페이지 (링크로 접속)
│   └── admin.html      # 관리자 페이지
└── functions/
    ├── _utils.js
    └── api/
        ├── config.js           # 공개 설정 조회 (투표 페이지용)
        ├── vote.js             # 로그인 확인 + 투표 제출 (POST 전용)
        └── admin/
            ├── login.js        # 관리자 로그인 확인
            ├── config.js       # 설정 저장/조회
            ├── members.js      # 회원 생성/조회/삭제
            └── results.js      # 결과 집계 조회 + 집계 초기화
```
